1. Project Overview

What is FarmLink?

FarmLink is a modern web application that connects farmers directly with buyers. It's designed like a mobile app running in a browser, allowing:

  • Buyers to search and purchase fresh produce directly from nearby farms
  • Farmers to list their products and manage orders
  • Instant Messaging between farmers and buyers
  • Multi-Language Support (English, Hindi, Marathi, Tamil, Telugu)
  • Multi-Currency Support (USD, INR, EUR, GBP)
  • Real-Time Location Tracking of nearby farms

Key Goals

Goal How It's Achieved
Reduce middlemen Direct farmer-to-buyer connection through chat
Support local agriculture Show farms within 10km radius
Easy to use Mobile-like interface with simple navigation
Inclusive Support multiple languages and currencies

4. How the App Works - The Big Picture

USER OPENS APP ↓ App.js initializes React Router ↓ FarmMarket.jsx loads as main component ↓ Header appears (logo, avatar, role toggle) ↓ Based on role (Buyer/Farmer), show appropriate content ↓ Bottom Navigation tabs appear ↓ User clicks tabs to navigate between pages ↓ Each interaction updates React State ↓ UI automatically re-renders with new data

The Two Roles:

👤 BUYER Role

Can:

  • Browse and search products from farms
  • Add items to cart
  • Place orders
  • Chat with farmers
  • View products on map

Cannot: Add or edit product listings

🌾 FARMER Role

Can:

  • Add new product listings
  • Manage existing listings
  • View orders
  • Chat with buyers
  • See analytics (orders, earnings)

Sees: Dashboard with stats instead of product grid

5. FarmMarket.jsx - The Main Component

This is the heart of your app. It's responsible for:

  • Managing all global state (data)
  • Handling all business logic
  • Coordinating between all components
  • Managing navigation and routing

Understanding State

State is like the app's memory. It stores data that changes over time.

// Buyer's shopping cart const [cart, setCart] = useState([]); // What products are being shown const [products, setProducts] = useState(INITIAL_PRODUCTS); // Currently selected product for detail view const [selectedProduct, setSelectedProduct] = useState(null); // Active navigation tab (home, map, cart, profile, messages) const [activeNav, setActiveNav] = useState("home"); // User's selected language const [language, setLanguage] = useState("en"); // User's selected currency const [currency, setCurrency] = useState("INR");

Key Functions in FarmMarket.jsx

Function What It Does Example
addToCart() Adds a product to the shopping cart User clicks + button on product
sendMessage() Sends a chat message to a farmer/buyer User types message in chat
handleAddListing() Farmer adds a new product listing Farmer fills form and clicks "List"
formatPrice() Converts prices to selected currency $2 becomes ₹165 (INR)
openConvo() Opens a conversation with a user User clicks on a message thread

How Data Flows After User Action

USER CLICKS BUTTON ↓ onClick event fires ↓ Function is called (e.g., addToCart()) ↓ State is updated (e.g., setCart()) ↓ React detects state change ↓ Component re-renders with new data ↓ User sees updated UI

💱 Multi-Currency System

How It Works:

// Currency definitions const CURRENCIES = { USD: { symbol: "$", rate: 1, name: "US Dollar" }, INR: { symbol: "₹", rate: 82.5, name: "Indian Rupee" }, EUR: { symbol: "€", rate: 0.92, name: "Euro" }, GBP: { symbol: "£", rate: 0.79, name: "British Pound" }, }; // Format price function const formatPrice = (priceInUSD) => { const currencyObj = CURRENCIES[currency]; const convertedPrice = (priceInUSD * currencyObj.rate).toFixed(2); return `${currencyObj.symbol}${convertedPrice}`; }; // Example: formatPrice(2.5) when currency is INR // → "₹206.25"

Logic:

  1. All prices stored in USD internally
  2. User selects preferred currency
  3. Multiply by exchange rate
  4. Display with correct symbol

🛍️ Shopping Cart Logic

State Management:

// Array of cart items const [cart, setCart] = useState([]); // Set to track which product IDs are in cart const [cartSet, setCartSet] = useState(new Set()); // Add to cart function const addToCart = useCallback((product) => { // Check if already in cart if (cartSet.has(product.id)) return; // Add to arrays setCart(c => [...c, product]); setCartSet(s => new Set([...s, product.id])); // Show success message showToast(`${product.emoji} Added to cart!`); }, [cartSet]);

Why Two Data Structures?

  • cart array - access all product details
  • cartSet - fast lookup (O(1) instead of O(n))
  • Together they're efficient and complete

# Navigate to project folder cd farmlink # Install dependencies npm install # Start development mode npm run dev # Open in browser # http://localhost:3000

What `npm run dev` does:

npm run dev ↓ Starts two servers: ↓ 1. JSON Server (port 4000) - Serves mock database - Listens at http://localhost:4000 ↓ 2. React Server (port 3000) - Serves React app - Listens at http://localhost:3000

Debugging Tips

⚠️ Common Issues & Solutions:
  • Port 3000 already in use: Kill the process: Get-Process node | Stop-Process -Force
  • Blank page loading: Check browser console (F12) for errors
  • API not working: Make sure JSON server is running on port 4000
  • Styles not loading: Clear cache (Ctrl+Shift+R) or hard refresh
  • Navigation not working: Check React Router setup in App.js

📋 Quick Reference Summary

App Architecture Overview

┌─────────────────────────────────────────┐ │ App.js (Router Setup) │ │ Sets up React Router and main routes │ └────────────────┬────────────────────────┘ │ ↓ ┌─────────────────────────────────────────┐ │ FarmMarket.jsx (Main Component) │ │ Manages all state & business logic │ │ Coordinates all child components │ └────┬──────────┬──────────┬──────────┬───┘ │ │ │ │ ↓ ↓ ↓ ↓ Header ProductCard CartModal ChatScreen Profile MapView Login etc.

Data Flow Pattern

User Interaction ↓ Event Handler Called ↓ State Updated (setCart, setProducts, etc) ↓ React Re-renders Components ↓ New Data Passed to Children as Props ↓ UI Updates on Screen

Component Props Pattern

// Parent (FarmMarket.jsx) passes data to child <ProductCard product={product} cartSet={cartSet} addToCart={addToCart} onSelect={setSelectedProduct} /> // Child (ProductCard.jsx) receives and uses props export default function ProductCard({ product, cartSet, addToCart, onSelect }) { return ( <div onClick={() => onSelect(product)}> {product.name} <button onClick={(e) => addToCart(product, e)}>+</button> </div> ); }

🎯 Key Takeaways for Beginners

✅ Do's

  • Break code into small, reusable components
  • Keep state in the highest parent component
  • Pass functions down as props for child-to-parent communication
  • Use meaningful variable names
  • Test in browser DevTools console
  • Read error messages carefully
  • Learn from other developers' code

❌ Don'ts

  • Don't mutate state directly (always use setState)
  • Don't put all code in one giant component
  • Don't forget to add keys to lists
  • Don't ignore console warnings
  • Don't hardcode data (fetch from API)
  • Don't start with complex features

💡 Pro Tips

  • Use React DevTools browser extension to inspect components
  • Break large components into smaller ones
  • Use descriptive naming for functions (handleClick, onAddToCart)
  • Document your code with comments
  • Test one thing at a time
  • Read the React documentation when stuck
  • Look at similar open-source projects
